# Folio Portfolio - Supabase Setup Guide

## Prerequisites
1. Create a Supabase account at https://supabase.com
2. Create a new Supabase project
3. Get your project URL and service role key

## Step 1: Set Up Environment Variables

Copy `.env.example` to `.env` and fill in your Supabase credentials:

```bash
cp .env.example .env
```

Then edit `.env`:
```dotenv
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key
PORT=4000
```

You can find these values in:
- **SUPABASE_URL**: Your project's API URL (Settings > API)
- **SUPABASE_SERVICE_ROLE_KEY**: Settings > API > Service Role (use this, not the anon key)

## Step 2: Create Database Tables

In the Supabase dashboard, go to the **SQL Editor** and run these commands:

### Create `sections` table:
```sql
CREATE TABLE sections (
  id BIGINT PRIMARY KEY DEFAULT (gen_random_bytes(8) || gen_random_bytes(8))::bigint,
  title TEXT NOT NULL,
  type TEXT NOT NULL,
  content TEXT,
  image_url TEXT,
  "order" INT NOT NULL,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);
```

### Create `portfolio_items` table (optional, for more detailed project management):
```sql
CREATE TABLE portfolio_items (
  id BIGINT PRIMARY KEY DEFAULT (gen_random_bytes(8) || gen_random_bytes(8))::bigint,
  title TEXT NOT NULL,
  description TEXT,
  image_url TEXT,
  link TEXT,
  tags TEXT[] DEFAULT ARRAY[]::TEXT[],
  "order" INT NOT NULL,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);
```

### Create `storage` bucket for images:
In Supabase Dashboard:
1. Go to **Storage**
2. Create a new bucket named `portfolio-images`
3. Make it public (set Read permissions)

Run this SQL in the Supabase SQL Editor to insert sample sections:

## Step 4: Upload Images to Supabase Storage

1. Go to Supabase Dashboard > Storage > `portfolio-images`
2. Upload your portfolio images
3. Copy the public URL from each image
4. Update the `sections` table with the image URLs:

```sql
UPDATE sections SET image_url = 'https://your-supabase-url/storage/v1/object/public/portfolio-images/your-image.jpg' WHERE id = 2;
```

## Step 5: Set Up Row-Level Security (Optional but Recommended)

For production, enable RLS on your tables:

1. Go to **Authentication > Policies**
2. Create a policy for `sections` table (allow public read)
3. Create an admin user for write operations

## API Endpoints

Once the server is running, you can use these endpoints:

### GET all sections
```bash
curl http://localhost:4000/api/sections
```

### GET specific section
```bash
curl http://localhost:4000/api/sections/1
```

### CREATE new section (requires authentication)
```bash
curl -X POST http://localhost:4000/api/sections \
  -H "Content-Type: application/json" \
  -d '{"title": "New Section", "type": "portfolio", "content": "...", "image_url": "...", "order": 8}'
```

### UPDATE section
```bash
curl -X PUT http://localhost:4000/api/sections/1 \
  -H "Content-Type: application/json" \
  -d '{"title": "Updated Title", ...}'
```

### DELETE section
```bash
curl -X DELETE http://localhost:4000/api/sections/1
```

## Deployment

When deploying to production:
1. Set environment variables in your hosting platform
2. Ensure `.env` is in `.gitignore`
3. Use service role key only on backend
4. Implement proper authentication before allowing edits
